What shipped — the four north-star surfaces

The user treats the terminal as read-only and acts through structured surfaces. Claude only ever sees its native channels (a tool result, or terminal stdin) — it spends no turn/token on panels and can't tell a phone tap from a keystroke.

✓ Conversation (read)

Durable turn-stream over the JSONL (stable ids, cursor+epoch, reset on compact/resume), collapsible tool-call/result cards. All content via textContent.

✓ Decision cards — Channel 1

Blocking PreToolUse hook holds the tool; permission / plan / question sheets answer allow/deny/choice straight back to the hook. Fail-closed, wait-for-human, trusted rendering.

✓ Input + interrupt — Channel 2

Composer injects to stdin only when idle_at_prompt (queued while busy); interrupt is the one immediate injection. Byte-exact, verified.

✓ Trusted, transparent

Decision packets render from structured tool_input in a trusted component (no agent HTML) — sidesteps the artifact-XSS / forged-approval path. Artifact panel height cap also removed.

Security hardening — the control plane is a remote input API

A 3-lab review treated the phone-facing control plane as a remote command-injection / exfil surface and found real holes in a first-cut Origin allowlist. All are closed and re-verified (7 dedicated origin tests):

FindingSevResolution
Wildcard *.ts.net/*.devtunnels.ms suffix trust (any tenant can provision a host → cross-tenant RCE)CriticalRemoved. Exact-pin to the server's OWN live origins (tunnelManager.publicUrl + meshManager), normalized by _originKey
Host-header same-origin (DNS-rebinding bypass on default-port / tunnel)CriticalRemoved. The client Host header is never read
Cross-origin GET read/exfil under --no-authImportantEvery control method is Origin-gated (reads too); no-Origin + same-origin still pass
Blanket RFC-1918 trust (lateral keystroke-injection on shared Wi-Fi)ImportantReplaced with the server's OWN interface addresses (os.networkInterfaces())
POST-only gating (future PUT/DELETE unprotected)ImportantGate keyed on CSRF-unsafe method, not literal POST

No-Origin callers (the decision hook, curl, native apps) always pass — Channel 1 is unaffected. Auth (Bearer/token) remains the primary gate; Origin is defense-in-depth.

Where it might fail — honest caveats

On-device dimensions (the open gate). No iOS simulator exists on Windows; Playwright-WebKit verifies logic, not real-Safari layout. Final safe-area / keyboard / dvh polish converges on the real iPhone over the mesh/tunnel — your parity pass. Landscape UI (keyboard eats height) is a device-loop item; its logic is orientation-agnostic and covered on portrait.
Opt-in by design. Mobile mode activates only via ?mobileMode=1 (or the cc-mobile-mode sessionStorage flag). Legacy touch-driving remains the default — a deliberate safe rollout, not a regression.
Non-tunnel public deploys. With --tunnel/--mesh the origin auto-pins (no config). If you expose ai-or-die on a public host without them, set AIORDIE_ALLOWED_ORIGINS=https://your-host, or same-origin browser POSTs from that host are gated.
Startup window (self-healing). A WS/POST in the sub-second gap after the tunnel routes but before Node scrapes its public URL is denied and recovers on the client's reconnect. You cannot reach the tunnel host before it exists, so there is no pre-tunnel exposure.
Coverage tail (by design). Folder-trust prompts and a script's own in-shell y/N inside a Bash tool are not hook-interceptable → raw-terminal fallback (rare). Benign MaxListeners/DEP0190 warnings in test output are pre-existing and non-blocking.
